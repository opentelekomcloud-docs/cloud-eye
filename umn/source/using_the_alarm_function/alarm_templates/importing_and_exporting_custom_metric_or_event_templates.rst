Importing and Exporting Custom Metric or Event Templates
========================================================

If you want to quickly create a metric or event template using an existing custom template, you can export the target template, modify the content as needed, and then import the modified template. This section describes how to import and export custom metric or event templates.

Constraints
-----------

Monitoring metrics and events vary by Cloud Eye version. Importing custom metric or event templates across regions with different versions may result in errors.

Importing Custom Metric or Event Templates
------------------------------------------

#. Log in to the management console.

#. In the upper left corner, select a region and project.

#. Choose **Service List** > **Cloud Eye**.

#. In the navigation pane on the left, choose **Alarm Management** > **Alarm Templates**.

#. Click the **Custom Templates** or **Custom Event Templates** tab.

#. Click **Import**.

#. Upload a JSON file, enter a template name, and click **OK**.


   .. figure:: /_static/images/en-us_image_0000001789425696.png
      :alt: **Figure 1** Import Template

      **Figure 1** Import Template

.. note::

   The template name must be different from that of a default one.

Exporting Custom Metric or Event Templates
------------------------------------------

#. Log in to the management console.
#. In the upper left corner, select a region and project.
#. Choose **Service List** > **Cloud Eye**.
#. In the navigation pane on the left, choose **Alarm Management** > **Alarm Templates**.
#. Click the **Custom Templates** or **Custom Event Templates** tab.
#. Locate the custom template and choose **More** > **Export** in the **Operation** column.
